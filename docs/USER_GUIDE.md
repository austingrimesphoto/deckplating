# Deckplating User Guide

Deckplating is an unofficial open-source prototype. It is not approved by the Department of the Navy or Department of Defense. Do not use it for CUI, classified information, sensitive operational data, counseling notes, case management, medical information, incident details, family information, home addresses, phone numbers, dates of birth, passphrases, setup codes, sensitive locations, or official records. Do not use it on government-furnished equipment or government networks unless authorized by local IT/N6.

Hosted guide: <https://deckplatingsetup.netlify.app/user-guide.html>

Live demonstration app: <https://deckplating.netlify.app>

## About Deckplating

Deckplating helps a Religious Ministry Team see routine coverage: where the team has been, where coverage has gone stale, and what broad unit or public/general location may need attention next.

It is for chaplains, Religious Program Specialists, RMT leads, and local teams responsible for many departments, divisions, tenant commands, or worksites.

It is not a counseling record, case-management system, medical tracker, official record, CUI system, classified system, or approved operational Navy application unless authorized.

## Workspace Selection

Use the approved workspace slug or workspace link from the operator/local lead. This is not open signup.

Never paste setup codes or passphrases into feedback forms, screenshots, public docs, or chat.

## Workspace Activation

Activation is for an approved controlled demonstration workspace. Enter the one-time setup code only in the activation field, confirm the workspace display name, select a public/general map center, and set the local admin passphrase.

## Choose Name and PIN

Select your team member display name and create a PIN. Display names may use practical workflow identity such as rank/last name or role/name. Do not enter phone numbers, dates of birth, family details, private contact information, or sensitive identifiers.

## Check In

Use `Locate Me` near a mapped public/general location, select the unit or units visited, and submit the check-in. A check-in stores generic visit metadata only: unit/location, timestamp, visitor display name, and coverage status.

Do not enter notes, counseling information, medical details, incident details, or case-management information.

## Manual Check-In

Use manual check-in when GPS is unavailable or the exact location should not be mapped. Sensitive spaces should remain unmapped.

## Coverage Board

Use Coverage to search and filter by command, department, broad area, public/general location, status, or visitor display name. Coverage is for planning routine presence, not ranking people or creating official records.

## Map

Map only public/general locations already broadly identifiable. Do not map SCIFs, restricted rooms, residences, deployed operational locations, or sensitive operational spaces.

## Mission Board

Mission Board recognition is a lightweight demonstration feature for coverage awareness. It should not shame or target named people, groups, faith communities, commands, or roles.

## Admin

Admin is for local setup and correction:

- Create broad areas.
- Create public/general locations.
- Create non-sensitive units, departments, divisions, or tenant-command labels.
- Create team member display names.
- Correct visit unit, member, or timestamp.
- Void accidental or duplicate check-ins.

Admin is not for counseling notes, case management, medical details, incident details, official records, or sensitive operational reporting.

## Feedback

Use the feedback form only for non-sensitive demonstration feedback. Do not include setup codes, passphrases, screenshots with production data, counseling details, CUI, medical details, incident details, personal information, sensitive locations, or official records.

## Safe-Use Boundaries

Allowed:

- non-sensitive workspace or command label
- broad area names
- public/general buildings or locations
- non-sensitive unit/department/division/tenant-command names
- team display names limited to practical identity
- generic visit timestamps
- generic coverage status

Prohibited:

- CUI or classified information
- counseling notes or case-management data
- medical or incident details
- family information
- home addresses, phone numbers, dates of birth, private email addresses
- setup codes or passphrases in screenshots, feedback, reports, or public docs
- sensitive operational locations, SCIFs, restricted rooms, deployed/theater locations
- official records

When uncertain, leave the information out and use command-approved channels outside Deckplating.
