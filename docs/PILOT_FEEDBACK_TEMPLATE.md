## Setup

- Did the setup guide make sense?
- Where did setup slow down?
- Which account step was hardest: GitHub, Supabase, or Netlify?
- Were the SQL steps clear?
- Were environment variables clear?
- Could the team complete setup without developer help?
- Did the team understand who locally owns GitHub, Supabase, and Netlify?

## First Use

- Could users select their name and create a PIN?
- Did anyone get stuck signing in?
- Did Add to Home Screen work?
- Did the app fit properly on the phone screen?
- Did the selected workspace/setup feel clear for a local owner?

## Check-In Workflow

- Did Locate Me work reliably?
- Did GPS accuracy cause problems?
- Did manual check-in make sense?
- Could users select the correct units at one location?
- Did confirmation appear quickly enough?
- Did Undo behave as expected?

## Offline Workflow

- Did the app open after one online launch?
- Did cached coverage appear offline?
- Did offline check-ins save on device?
- Did queued visits survive closing and reopening the app?
- Did sync work when connectivity returned?
- Did users understand the sync status bar?
- Did any offline behavior make the team hesitant to trust the app?

## Admin Workflow

- Was it clear how to add areas, locations, units, and team members?
- Was the location mapping notice clear?
- Did Activity Log corrections make sense?
- Did voided entries behave as expected?
- Could the pilot lead manage the app without developer intervention after setup?

## Coverage And Mission Board

- Did the Coverage Board help identify where to go next?
- Did command/unit detail drawers appear where expected?
- Did Mission Brief nudges feel useful or distracting?
- Which tone was used: Professional, Friendly, or Deckplate Banter?
- Did Mission Board reward meaningful coverage rather than raw volume?
- Were badges understandable?

## Reports

- Could users find referral and confidential care reporting?
- Were the counts useful?
- Was it clear that indicators are generic location-level counts only?
- Did the reports help local leaders explain ministry coverage without storing sensitive details?

## Safe Use

- Did users understand what not to enter?
- Did anyone try to enter counseling, medical, personal, or sensitive operational details?
- Were mapped locations appropriate and publicly identifiable?
- Did any screen or wording create confusion about the safe-use boundary?
