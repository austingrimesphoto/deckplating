Self-hosted RMT setup

Start here. Do these steps in order.

This page walks you through making your own Deckplating app. Keep this tab open. Open each button in a new tab, finish that task, then come back here for the next step.

Do not skip ahead. You need accounts first, then your app copy, then the database, then Netlify.
Open User Guide Open Feedback Form Open Repository

Step 1

Start with GitHub

Start by creating or opening your GitHub account. Then use GitHub sign-in when Supabase and Netlify offer it. That keeps the setup simpler and makes turnover easier.

Use the name Deckplating everywhere normal users will see it. Recommended project names: GitHub repository deckplating, Supabase project deckplating, and Netlify site deckplating or deckplating-your-command.
Create GitHub Account Open GitHub Login Open Supabase Sign Up Open Netlify Sign Up
Save these logins somewhere approved. Do not make the whole setup depend on one person’s personal account.

Step 4

Generate the secret settings for Netlify

Supabase gives you two values Netlify needs. Paste them here. This runs only in your browser and does not send values anywhere.

Step 5

Deploy your app to Netlify

Netlify turns your GitHub copy into a live website. Click Import an existing project, choose GitHub, add the values from Step 4, then deploy.

Open Netlify Import

Step 6

Launch and finish setup

When Netlify says the deploy is published, open the site URL. Then finish your real local setup inside the app.

Step 7

Send pilot feedback back to the app owner

Use this form during or after the pilot. Submissions are stored by Netlify Forms on this setup site, so the app owner can review them without copy and paste.

Keep feedback unclassified and non-sensitive. Do not include counseling details, medical details, home addresses, personal identifying information, or sensitive operational locations.

Direct feedback link for pilots: https://deckplatingsetup.netlify.app/#feedback
