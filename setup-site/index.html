About Deckplating

Coverage awareness for Religious Ministry Teams

Deckplating helps an RMT answer a practical question: where is ministry presence happening, and where is it missing? It gives the team a shared picture of routine deckplate coverage without turning the app into a counseling record or case file.

What it is

A mobile web app for unclassified, non-sensitive coverage tracking by workspace, location, unit, and visit.

Who it is for

Command chaplains, Religious Program Specialists, local RMT leads, and teams responsible for many departments, divisions, tenant commands, or worksites.

What it provides

Quick check-ins, coverage status, search, map views, Mission Board recognition, admin correction tools, activity history, and safer handoff data for the next watch team.

Why it matters

Busy teams can unintentionally revisit familiar spaces while quiet or scattered units go unseen. Deckplating helps plan rounds by actual coverage need and protects sensitive ministry details by keeping them out of the system.

How demonstration onboarding works

Manual approval first, then local setup

1. Request

Submit the workspace request form with non-sensitive command and lead information.

2. Approval

The operator reviews the request, creates the workspace, and issues a one-time setup code.

3. Local setup

The approved lead opens the workspace link, enters the setup code, sets the local admin passphrase, and creates roster/map data.

Workspace request

Request controlled demonstration access

Requests enter the Deckplating operator approval queue. If approved, the lead may receive the workspace link, setup guide, and first steps at the administrative contact email provided here. A setup code is sent only when the operator intentionally includes it.

After approval

What the local lead does next

Approved lead package

The operator provides the workspace link on deckplating.netlify.app, the user guide, and setup information. The setup code is included only when the operator intentionally sends it to the administrative contact email.

  1. Open the workspace link from the operator.
  2. Select Activate workspace.
  3. Enter the one-time setup code.
  4. Confirm the workspace display name and installation/map center.
  5. Set the local admin passphrase for this workspace.
  6. Continue to local setup and create areas, locations, units, and team members.
  7. Send the same workspace link to team members after the roster is ready.
