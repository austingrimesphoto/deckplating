Phone-first field guide

How to use Deckplating without guessing.

This guide is for chaplains, RPs, and local leads after the app is already set up. It starts with the basic daily workflow, then walks through each screen, correction path, and reporting use case.

Best first lesson

Teach new users five things first: sign in, Locate Me, manual lookup, check-in confirmation, and the sync bar.

Safe use still applies

Use Deckplating only for unclassified, non-sensitive coverage metadata. No counseling notes, medical details, or sensitive locations.

Quick Start

Ten-minute user version

1. Sign in

Select your roster name and enter your 4-digit PIN.

2. Find the location

Tap Locate Me. If GPS is not usable, use manual lookup.

3. Select units

Choose one or more units at that mapped location, or one unmapped unit manually.

4. Submit

Tap Check In or Submit Manual Check-In.

5. Review

Check the confirmation details, points, and undo option.

6. Move on

Tap Done. The app should not slow down your actual visit rhythm.

Philosophy

What the app is for

Deckplating is built around practical presence. The point is not to produce a perfect digital record. The point is to help a Religious Ministry Team see where ministry presence is happening, where it is stale, and where it needs to go next.

That is why the app emphasizes meaningful coverage, easy correction, safe-use limits, and a very short check-in flow.

Not the mission
  • Not a counseling record
  • Not case management
  • Not official recordkeeping
  • Not a place for CUI or sensitive operational data

Check In

The main working screen

Annotated screen diagram based on the current phone workflow.

  1. Locate Me tries to match your phone position to a saved mapped location.
  2. Select one or more units only from that one physical location.
  3. Submit the visit. If GPS is unavailable, use Manual unit lookup.
  4. The confirmation screen is the correction window. Undo there if the visit was accidental.
Optional visit indicators

These are generic counts only. They are not tied to a person, department, counseling case, or referral detail.

If Locate Me fails

Check precise location permission, try again, then fall back to manual lookup. GPS failure should never stop ministry tracking.

Coverage

Where attention should go next

Coverage Board is the main planning view. It sorts the work toward never-visited, overdue, and due-soon units, and it lets you drill into a command directly where you tapped it.

  • Use filters to narrow by area, type, overdue, never, or dates.
  • Tap a command card to open its detail drawer directly under that card.
  • Use the report section to review referral and confidential-care totals by mapped location over time.
  1. 1. Team-level status summary and top needs.
  2. 2. Tap the command card you care about.
  3. 3. Detail opens right there instead of forcing a long scroll.

Map

Mapped public locations and offline fallback

  1. Online mode shows the map and status circles for mapped public locations.
  2. Each location card shows area, radius, and attached unit count.
  3. Offline mode drops the map tiles and keeps the cached location list available.
Important limit

Offline map view is intentionally simple. The list remains usable even when map tiles are unavailable.

Mission Board

Recognition for meaningful coverage

Mission Board keeps score as the main ranking metric, but it shows more useful context than raw visit count: meaningful visits, distinct units, recovered units, active days, and badges.

First Rounds

First qualifying check-in this month.

Recovery Team

Recovered an overdue or never-visited unit.

Gray to Green

Completed a first-ever visit.

Wide Coverage

Visited five distinct units this month.

Sustained Presence

Worked four distinct days this month.

Coverage Sweep

Helped clear overdue/never units from an area.

What users should understand
  • The app rewards recovery, breadth, and consistency.
  • One meaningful recovery is worth more than repetitive check-ins at an already current space.
  • Badges should encourage action, not pressure people to game the board.

Admin

Local lead tools

Locations

Create mapped public locations, adjust coordinates, set radius, and attach units.

Activity Log

Filter check-ins, correct unit/member/date mistakes, and soft-void invalid entries.

Settings

Set Mission Brief tone: Professional, Friendly, or Deckplate Banter.

Mapping rule

Map only publicly identifiable buildings or general areas. If there is any doubt, leave the location unmapped and use manual check-in.

Workflows

Most common use cases

Basic daily round

  1. Open the app.
  2. Read the Mission Brief.
  3. Locate the current building.
  4. Select the unit or units there.
  5. Check in.
  6. Tap Done and move on.

Offline check-in

  1. Open the app after it has previously launched online.
  2. Use cached coverage data.
  3. Submit the visit.
  4. Look for a status like Saved on this device or pending upload.
  5. Reconnect later and use Sync Now if needed.

Fixing a mistake

  1. If the error was immediate, use Undo this check-in.
  2. If the check-in is older or more complex, use Admin > Activity Log.
  3. Edit only what needs correction. Voided records stay in history and stop affecting score and coverage.

Teaching a new teammate

  1. Start with one mapped location and one manual check-in.
  2. Show the confirmation screen and undo.
  3. Show the sync bar meanings.
  4. Then show Coverage and Mission Board.
